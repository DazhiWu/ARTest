# 智能管线定位流程 - 完整使用指南

## 🎯 概述

本系统实现了精确的5步流程控制，确保地下管网与真实地面的完美贴合：

1. **摄像头图像采集** → 2. **地面识别与特征提取** → 3. **三维平面生成与验证** → 4. **GPS定位数据获取** → 5. **三维管线精确渲染**

## 📋 详细步骤说明

### 步骤 1/5: 摄像头图像采集 📷

**目标**：通过设备摄像头实时采集地面图像数据，捕捉地面轮廓、纹理及特征点信息

**执行内容**：
- 检查摄像头是否已启动
- 自动启动摄像头（如需要）
- 捕获当前帧图像数据
- 保存图像特征信息

**视觉指示**：
- 步骤图标高亮橙色
- 显示"采集中..."状态
- 详情面板显示采集进度

---

### 步骤 2/5: 地面识别与特征提取 🔍

**目标**：基于采集的图像数据进行精确地面识别

**执行内容**：
- **优先方式**：WebXR (ARCore/ARKit) 平面检测
  - 利用设备原生AR能力
  - 精确检测真实物理平面
  - 捕捉特征点与轮廓信息
  
- **回退方式**：手动检测
  - 手动点击地面位置
  - 射线检测虚拟平面
  - 视觉反馈确认位置

**视觉指示**：
- 地面网格逐步显现
- 显示"识别中..."状态
- 详情面板显示检测到的特征信息

---

### 步骤 3/5: 三维平面生成与验证 📐

**目标**：基于识别结果生成与实际地面完全一致的三维渲染平面

**执行内容**：
- 生成三维平面网格
- 匹配地面尺寸、形状
- **稳定性验证（10帧检查）
- 确保平面与物理地面精确匹配

**验证标准**：
- 连续10帧位置稳定
- 平面姿态（法线一致
- 用户视觉确认

**视觉指示**：
- 绿色网格显示在真实地面位置
- 进度条动画显示验证进度
- 详情面板显示稳定性检查计数

---

### 步骤 4/5: GPS定位数据获取 📍

**目标**：获取手机的GPS定位信息，包括经纬度、海拔及方向数据

**执行内容**：
- 获取精确GPS坐标
- 读取海拔高度数据
- 检测设备朝向角度
- 记录定位精度信息

**数据内容**：
```javascript
{
    latitude: 39.9042,    // 纬度
    longitude: 116.4074,  // 经度
    altitude: 50,         // 海拔
    heading: 45,          // 朝向角度
    timestamp: 1234567890  // 时间戳
}
```

**回退方案**：
- GPS不可用时使用模拟位置
- 用户可手动输入坐标

---

### 步骤 5/5: 三维管线精确渲染 🔧

**目标**：根据GPS定位信息在已生成的三维平面上精确渲染三维管线模型

**执行内容**：
- 设置坐标原点
- 加载管线数据
- 应用地理坐标转换
- 对齐管线到地面
- 应用埋深参数
- 最终渲染显示

**关键技术**：
- WGS84 → 局部笛卡尔坐标转换
- 地面高度精确对齐
- 管线埋深物理模拟
- 真实比例1:1渲染

## ⚠️ 常见问题

### Q: 流程卡在某一步骤？

**A**: 
1. 检查设备权限是否正常
2. 查看详情面板的错误信息
3. 尝试重新开始流程
4. 确保光线充足，地面纹理丰富

### Q: 地面识别失败？

**A**:
1. 尝试手动点击地面指定位置
2. 确保光线充足
3. 选择有明显纹理的地面（地砖、水泥地等）

### Q: 管线位置不对？

**A**:
1. 检查GPS定位精度
2. 使用位置校准功能微调
3. 重新执行流程获取更精确的地面

### Q: WebXR不可用？

**A**:
系统会自动回退到手动模式
1. 手动点击地面
2. 系统提示操作
3. 完成后续步骤
